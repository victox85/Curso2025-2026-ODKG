Dataset Requirements

Group: 09 · Members: Andosilla Herráiz, Lucas — Gil Planes, Germán — Beamonte González, Sergio — Herrero Míguez, Paula — Hunziker Cadabal, Celina

Selected datasets

  1. City of Melbourne — Pedestrian Counting System (Monthly counts per hour)

    Identifier: pedestrian-counting-system-monthly-counts-per-hour

    Updated: monthly

    Theme: Transport

    Language: English

    Key columns: id, location_id, sensing_date, hourday, pedestriancount, sensor_name, location.

    Hourly pedestrian counts since 2009; mergeable with “Sensor Locations” via sensor_id/location_id.
  2. City of Melbourne — Pedestrian Counting System · Sensor Locations

    Identifier: pedestrian-counting-system-sensor-locations

    Updated: daily

    Theme: Transport

    Key columns: location_id, sensor_name, status, latitude, longitude, location.

    Provides geographic reference for pedestrian sensors. Joinable to counts via location_id.
  3. Pedestrian Sensor Locations (Enriched)

    Identifier: pedestrian-counting-system-sensor-locations_enriched

    Theme: Transport / Urban context

    Key columns: location_id, sensor_name, status, latitude, longitude, street, housenumber, postcode, city, country, formatted, latitude, longitude.

    Enhances the original “Sensor Locations” dataset with geocoded address fields (street, postcode, city, etc.) obtained via API, enabling precise spatial linkage with microclimate data and administrative boundaries.
  4. Public holidays in Australian capital cities (2004–2023)

    Identifier: public-holidays

    Theme: Events / Government

    Key columns: Date, Public holiday, Melbourne, National_holiday.

    Lists official public holidays for all Australian capital cities (2004–2023). Allows temporal linkage with pedestrian and microclimate data to study holiday-related patterns.
  5. Microclimate Sensor Locations

    Identifier: microclimate-sensor-locations_enriched

    Theme: Environment / Climate

    Key columns: Site_ID, GateWayHub_ID, Site_Status, Last_Data_Local, Latitude, Longitude, Location.

    Lists active microclimate sensor sites in the City of Melbourne with status, coordinates and last data date. Enables spatial linking with pedestrian sensors and weather readings.
  6. Microclimate Sensor Locations (Enriched)

    Identifier: microclimate-sensor-locations_enriched

    Theme: Environment / Climate

    Key columns: Site_ID, GateWayHub_ID, Site_Status, Last_Data_Local, Latitude, Longitude, Location, street, housenumber, postcode, city, country, formatted.

    Extends the base microclimate sensor locations with address and geocoding fields retrieved via API, improving linkage with pedestrian and administrative datasets.
  7. Microclimate Sensors Data

    Identifier: microclimate-sensors-data

    Theme: Environment / Weather

    Key columns: Device_id, Time, SensorLocation, AirTemperature, RelativeHumidity, AverageWindSpeed, PM25, Noise.

    Provides detailed environmental readings from each microclimate sensor, linkable to sensor locations via Device_id/SensorLocation for spatial and temporal analysis.

Requirements assessment (R1–R6)

Req.What it asksStatusEvidence / comments
R1 Smart-city domain Meets All datasets belong to smart-city domains: Transport (pedestrian counts, sensors), Environment (microclimate), and Events (public holidays). They support mobility, climate, and planning analyses.
R2 Available as CSV Meets All datasets are provided or exported as CSV files, enabling easy integration and processing.
R3 Open license Meets Datasets from the City of Melbourne portal are released under Creative Commons Attribution 4.0 (CC BY 4.0). The “Public Holidays” dataset is under Creative Commons CC0 1.0 (Public Domain). All datasets are therefore open, reusable, and publishable.
R4 Linkable to generic entities Meets Linking keys: sensor_id/location_id (pedestrian ↔ locations), date (↔ holidays), and geospatial coordinates (↔ microclimate sensors, districts, POIs). Enriched versions add address-level attributes to facilitate linkage to generic real-world entities.
R5 Documentation (optional) Meets Each dataset includes metadata, data dictionaries, or field descriptions on the open-data portal or accompanying notes.
R6 Multiple sources (optional) Meets Data integrate multiple smart-city sources: pedestrian counts, enriched location data, climate sensor networks, and public-holiday events, enabling multi-domain analytics (mobility ↔ weather ↔ events).

Entities in the selected datasets that can be linked

  • Locations: latitude/longitude, street, postcode → link to districts or Wikidata POIs.
  • Sensors: sensor_id/location_id, sensor_name, type.
  • Time: sensing_date, hourday, timestamp → link to public holidays or external events.
  • Weather variables: temperature, humidity, wind speed, particulate matter, noise — joinable by time and location with pedestrian data.

Entities in other datasets that can be linked to ours

  • Wikidata / DBpedia: POIs, squares, and districts corresponding to sensor locations.
  • Geonames: geographic hierarchy (city, suburb, postcode).
  • Australian Government APIs: official holidays and events datasets.
  • Open weather or climate APIs: to validate or enrich microclimate sensor data.

Scope note: The document covers dataset identification and evaluation under R1–R6. Enriched datasets enable more complete spatial linking across mobility, environment and event data.
