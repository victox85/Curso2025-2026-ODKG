Application Requirements & Mock-ups (Assignment 2)

Scope for this deliverable: select datasets and specify the application idea with UI mock-ups.

Selected datasets

  • Pedestrian Counting System — Monthly counts per hour: hourly pedestrian totals since 2009; joinable with sensor location data via sensor_id/location_id.
  • Pedestrian Counting System — Sensor Locations: coordinates, status, and metadata for each sensor.
  • Pedestrian Sensor Locations (Enriched): extends the original locations dataset with address-level fields (street, postcode, etc.) obtained from an API.
  • Microclimate Sensor Locations: base dataset listing microclimate sensors with coordinates and operational status.
  • Microclimate Sensor Locations (Enriched): adds geocoded address columns (street, city, postcode, etc.) to ease linkage with other spatial datasets.
  • Microclimate Sensors Data: detailed environmental readings (temperature, humidity, wind, PM, noise) from each sensor, linked via Device_id.
  • Public Holidays: dates of official holidays in Melbourne; used to analyse pedestrian and weather patterns on holidays vs. regular days.

Why these datasets fit the assignment (R1–R6)

  • R1 — Smart city domain: datasets cover mobility, environment and events — key smart-city dimensions that interact (pedestrian movement ↔ weather ↔ holidays).
  • R2 — CSV available: all seven datasets are downloadable or exportable in CSV format.
  • R3 — Open license: all datasets are CC BY 4.0 except Public Holidays, which is under CC0 1.0 (public domain).
  • R4 — Linkable: shared keys and coordinates enable joining: sensor_id/location_id (pedestrian data), Device_id/SensorLocation (microclimate), and date (public holidays). Enriched datasets provide address-level linkage to real-world entities.
  • R5 — Documentation (optional): all datasets include metadata or field descriptions on their open-data portals.
  • R6 — Multiple sources (optional): integrates data from several smart-city domains — transport, climate, and events — supporting multidimensional analytics.

Application concept

Working title: Urban Dynamics Explorer (Melbourne)

Goal: Analyse how pedestrian activity varies with weather and holidays, using linked data from sensors and public events.

Concrete questions the UI should answer

  • How do temperature, humidity, or wind affect pedestrian activity in different city areas?
  • Which zones are busiest during public holidays versus regular weekdays?
  • Can we identify patterns of comfort or congestion based on microclimate and mobility data?

Minimal feature list for the prototype

  • Load the seven CSV datasets locally and join them by time, location, and sensor identifiers.
  • Filter by date, time range, or weather conditions; aggregate to show averages and peaks.
  • Interactive map combining pedestrian intensity and microclimate readings, with tooltips and drill-down.
  • Comparator view: regular day vs. public holiday or different weather conditions for the same location.
