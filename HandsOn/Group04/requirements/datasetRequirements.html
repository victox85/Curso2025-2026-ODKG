R1. Smart-cities domain Satisfied

Both datasets are urban mobility data (origin–destination flows and zone definitions), a core smart-city vertical published as part of Spain’s national mobility open data initiative by the Ministry of Transport (MITMA).2,3

R2. Available as CSV Satisfied

The selected files are CSVs (.csv) suitable for transformation to RDF and publication.

R3. Open license for publication Satisfied

MITMA provides these mobility datasets under the Licencia de datos abiertos del MITMA (LDA MITMA), which allows reuse and redistribution (including transformed works like RDF) with attribution, preservation of metadata (e.g., update dates and license), and no implied endorsement.1,4

R4. Easily linkable to real-world entities Satisfied

  • From the selected datasets:
    • id_origin, id_destination ↔ zones.id (municipality INE-style codes) and zones.name — link to municipal entities.
    • date, hour — link to time entities (calendar/OWL-Time).
    • age, gender, income, activity_* — map to controlled vocabularies (SKOS concepts) if desired.
  • External datasets we can link to:
    • Wikidata / DBpedia / GeoNames: municipalities by INE code or label (e.g., Madrid “28079”).
    • INE administrative lists: official municipality codes; or OpenStreetMap features for municipal boundaries.

R5. Documentation exists (optional) Satisfied

MITMA’s Open Data Movilidad portal publishes format notes and a license PDF; pySpainMobility and companion tooling (e.g., spanishoddata) provide public docs describing fields, study areas, and access patterns.2,4,5,6

R6. Multi-source possibility (optional) Partially (potential)

We currently use two CSVs from the same publisher (MITMA). It is straightforward to add complementary sources later (e.g., municipal shapefiles, population/POIs, weather, GTFS), joining by municipality code or spatial joins.
