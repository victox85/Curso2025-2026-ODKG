Proposed application: Linked Universities Data Explorer

Web application to explore, filter, and link information about university degrees in Spain, built upon CSV datasets from regional open data portals (e.g., Andalucía, Aragón, Cantabria, Castilla y León, Castilla-La Mancha, Cataluña). It enables searching universities and degrees, visualizing metadata, filtering by region or field of study, and linking CSV entities (universities, faculties, degrees, municipalities) with external knowledge graphs such as Wikidata, DBpedia, and GeoNames.

1) Scope and Objectives

  • Discover available degrees and universities across Spanish regions.
  • Consult information about each degree (university, faculty, location, credits, knowledge area, admission score).
  • Filter by Comunidad Autónoma, university, branch of knowledge, or academic year.
  • Link degrees and institutions with external entities (Wikidata, GeoNames, esDBpedia).
  • Export filtered results as RDF (Turtle) and enable SPARQL queries over the generated graph.

2) Functional Requirements

IDDescriptionValidation
F1Search universities or degrees by text (name, university, location).At least one match appears for valid inputs.
F2Filter by Comunidad Autónoma, university, or branch of knowledge.Filters can be combined; results counter updates dynamically.
F3Degree detail page with metadata and outbound Linked Data URIs (sameAs / locatedIn).URIs are clickable and resolve to external data sources.
F4Export current results to RDF (Turtle).Download includes visible fields and linked URIs.
F5Provide local SPARQL endpoint over the generated RDF graph.At least 3 example queries return expected results.
F6Map visualization with university campus markers.Markers visible and accurately positioned via coordinates.

3) Non-functional Requirements

  • Accessibility: Contrast-compliant design, keyboard navigation (Tab, Enter), and alternative texts for visual content.
  • License compliance: Respect original dataset licenses (CC-BY or equivalent) and propagate them in exported RDF.
  • Internationalization: Interface in English; metadata fields maintain their original Spanish values.

Screen description

The screen lists university degrees with relevant metadata. The left panel provides text search and multiple composable filters. The right panel dynamically updates the list of results.

User interactions

  • Apply one or more filters and observe the result count.
  • Scroll through the list of matching degrees.
  • Click a degree to view its detailed record (see 4.3).
  • Switch to “Map” to view geographic distribution (see 4.2).

Screen description

The map visualizes university campuses or faculties from the selected datasets. Users can pan, zoom, and click markers to view linked degree details.

  • Click markers to open degree or university detail view.
  • Return to list view via “Degree Search” (see 4.1).
