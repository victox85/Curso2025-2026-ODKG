Data Quality Observations

  • Completeness: Some fields have missing values (e.g., district info for schools outside Madrid city, phone/fax numbers).

  • Temporal coverage: Foundation dates range from 1980s to 2021, indicating historical and recent schools.

2.3 License for Generated Linked Data

Chosen License: Creative Commons Attribution 4.0 International (CC BY 4.0)

Rationale:

  • Compatibility: CC BY 4.0 is compatible with all source licenses. It satisfies the attribution requirements of both the Comunidad de Madrid (explicit CC BY 4.0) and the Ayuntamiento de Madrid (Terms of Use requiring source citation).
  • Openness: Ensures the SafeSchool Madrid linked dataset remains open and reusable for research, public applications, and commercial purposes.
  • Standard: CC BY 4.0 is widely adopted in the Linked Open Data community and aligns with FAIR data principles (Findable, Accessible, Interoperable, Reusable).
  • Attribution statement: All generated RDF data will include provenance metadata (dct:source, dct:license, prov:wasDerivedFrom) linking back to the original datasets.

3. Resource Naming Strategy

This project uses persistent HTTP URIs under the domain http://safeschool.linkeddata.es/ with separate namespaces for ontology terms and data instances. The design follows Linked Data best practices (hash URIs for the ontology terms; slash URIs for instances with 303 content negotiation).

3.1 Namespaces

  • Ontology terms (safeschool:): http://safeschool.linkeddata.es/ontology#
  • Instances (resource:): http://safeschool.linkeddata.es/resource/
  • Controlled vocab (reserved) (vocab:): http://safeschool.linkeddata.es/vocab/

3.2 URI patterns

Ontology class/property: http://safeschool.linkeddata.es/ontology#Accident, ...#accidentType
Accident: http://safeschool.linkeddata.es/resource/Accident-{num_expediente} (e.g., .../Accident-2025S000056)
Educational center: http://safeschool.linkeddata.es/resource/EducationalCenter-{centro_codigo} (e.g., .../EducationalCenter-28000029)
Fixed speed camera: http://safeschool.linkeddata.es/resource/SpeedCamera-{id} (e.g., .../SpeedCamera-1)
Municipality: http://safeschool.linkeddata.es/resource/Municipality-{codigo} (e.g., .../Municipality-005)
District: http://safeschool.linkeddata.es/resource/District-{codigo} (e.g., .../District-05)
Road: http://safeschool.linkeddata.es/resource/Road-{slug} (e.g., .../Road-M30)
Location: http://safeschool.linkeddata.es/resource/Location-{context}-{id} (e.g., .../Location-Accident-2025S000056)

3.3 Persistence, normalization, and negotiation

  • Persistence: identifiers reuse stable source IDs (e.g., num_expediente, centro_codigo, radar number). URIs do not embed temporal or presentation details.
  • Normalization: strings are upper-camel for local names; road slugs use uppercase ASCII with hyphens (M-30, CALLE-LOPEZ-DE-HOYOS).
  • Content negotiation: the ontology doc at http://safeschool.linkeddata.es/ontology and instance docs at .../resource/* perform 303 redirects to HTML or RDF (Turtle/JSON-LD) based on the Accept header.
  • Licensing & provenance: dataset metadata include dct:license (CC BY 4.0), dct:source, and prov:wasDerivedFrom pointing to the original portals.
