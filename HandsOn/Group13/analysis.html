Project: Smart Mobility Dashboard for Cary, NC

Domain: Smart Cities - Transportation and Mobility

1. Data Source Analysis

1.1 Dataset Overview

Three CSV datasets from Cary Open Data portal are being used:

Dataset Description Key Entities Number of Rows
eco-counter-sites.csv Metadata about pedestrian/bicycle/vehicle counting sensor locations Sensor sites with geographical coordinates, names, and installation details 40
eco-counter-data.csv Time-series traffic count data from Eco-Counter sensors Traffic observations with counts, timestamps, and sensor references 623441
electric-vehicle-charging-stations.csv EV charging station information and usage data Charging stations with locations, capacity, and charging session data 20142

1.2 Schema Analysis

eco-counter-sites.csv

Columns:

  • id (integer): Unique identifier for each sensor site
  • site_active (boolean): Whether the site is active or not
  • site_owner (string): Owner of the counter site (Cary or CAMPO)
  • name (string): Name of the location
  • coordinates (string): Geographic coordinates as a string
  • user_type (string): Type of counter (pedestrian, bicycle, vehicle, undefined or no value)
  • installation_date (date): When the sensor was installed
  • counter (string): Unique identifier for each counter
  • photos (string (url)): URLs to the photos of the counters.

eco-counter-data.csv

Columns:

  • date (datetime): Date and time of the observation
  • day_of_week (string): Day of the week of the observation
  • id (integer): Reference to eco-counter-sites id
  • name (string): Reference to eco-counter-sites name
  • counts (integer): Number of pedestrians/bicycles/vehicles counted
  • site_active (boolean): Whether the site is active or not
  • user_type (string): Type of counter (pedestrian, bicycle, vehicle, undefined or no value)
  • installation_date (date): When the sensor was installed
  • photos (string (url)): URLs to the photos of the counters.
  • status (string): No information of this column, only value is "raw".
  • site_owner (string): Owner of the counter site (Cary or CAMPO)

electric-vehicle-charging-stations.csv

Columns:

  • start_date (datetime): Date and time when the vehicle started charging
  • station_name (string): Name of the station
  • charging_time_hh_mm_ss (string): Charging time expressed in hh:mm:ss
  • energy_kwh (int): Energy charged in kWh
  • address1 (string): Street address
  • address2 (string): Street address (part 2)
  • city (string): City (always "Cary")
  • state_province (string): State Province (always "North Carolina")
  • zip_postal_code (int): Zip postal code
  • fee (int): Fee (always 0)
  • model_number (string): Model number of the EV Charger
  • latitude (decimal): Geographic coordinate
  • longitude (decimal): Geographic coordinate
  • geopoint (string): Geographic coordinate

1.3 Data Quality Observations

  • Geographical coordinates allow linking to external geographical databases (GeoNames, DBpedia)
  • Temporal data enables time-series analysis and pattern identification
  • Hierarchical relationships exist between sites and their observations/sessions
  • Data contains both static entities (locations) and dynamic observations (counts, sessions)

3.3 Content Negotiation Strategy

URIs support content negotiation to provide multiple representations:

  • text/html: Human-readable HTML description of the resource
  • application/rdf+xml: RDF/XML serialization
  • text/turtle: Turtle serialization
  • application/ld+json: JSON-LD serialization
