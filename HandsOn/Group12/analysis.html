License Analysis of the Dataset

We have analyzed the license of the dataset available at data.gv.at (dataset id d99a5b1f…9517), following the defined methodology from the Chapter 3 of D4.1 deliverable from the READY4SmartCities project.

1. Identification of the rightsholder

The dataset is published on the Austrian open data portal data.gv.at. The metadata on the portal identifies the data provider / rightsholder as the City of Vienna (Magistratsabteilung 23 – Wirtschaft, Arbeit und Statistik).

2. License verification

The dataset is published under the Creative Commons Attribution 4.0 International (CC BY 4.0) license. This license permits copying, redistribution, adaptation and commercial reuse provided that appropriate attribution is given to the original source and a link to the license is included. See the license text: CC BY 4.0.

3. Implications for Linked Data generation

  • Since the dataset is publicly available under CC BY 4.0, it can be reused to generate Linked Data.
  • When publishing the RDF/Linked Data, the following conditions must be respected:
    • Attribution: clearly credit the City of Vienna as the source.
    • License metadata: include a reference to the CC BY 4.0 license in the RDF metadata (for example using dct:license or owl:license triples).
    • Derived data: indicate any modifications or transformations made to the original data.

Conclusions

At the time of this analysis the dataset is openly licensed (CC BY 4.0) and therefore suitable for reuse and Linked Data publication, provided that attribution and license metadata are included and that any privacy-sensitive content is handled appropriately.

Resource Naming Strategy

Purpose

This document explains the URI and resource naming strategy for our dataset. Since the dataset is quite small, hash URIs will be used (see Sauermann & Cyganiak, 2008).

The URI domain to be used is: https://iamabsa.github.io/chargingstation##12345/.

Ontology Model Classes

  • Classes follow the pattern: 
    https://iamabsa.github.io/ontology/<ontologyName>#<className>
  • Example: Charging station class: 
    https://iamabsa.github.io/ontology/ChargingOntology#ChargingStation

Ontology Model Properties

  • Properties follow the pattern: 
    http://iamabsa.github.io/ontology/<ontologyName>#<propertyName>
  • Example (the hasPower property of ChargingStation): 
    http://iamabsa.github.io/ontology/ChargingOntology#hasPower

Instances

  • Instances follow the pattern: 
    https://iamabsa.github.io/resource/<className>#<identifier>
  • Example (an instance of a charging station in Vienna): 
    https://iamabsa.github.io/resource/ChargingStation#WIEN1234
