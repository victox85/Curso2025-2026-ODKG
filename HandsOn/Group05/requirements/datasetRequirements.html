Requirements

  • R1. Data are in the smart cities domain.

    • The objective of this project is to obtain information related to cycling in Madrid, using datasets about bicycle stations, their trips, and the historical availability, in order to promote ecological transportation.

  • R2. Data are available as a CSV file.

    • All datasets obtained from their respective sources have been provided in CSV format.

  • R3. Data have an open license so it can be published.

    • The data are regulated by Article 3, section 2 of Law 37/2007, of November 16 and the Creative Commons Attribution License, meaning that they can be freely used as long as the original source is referenced.

  • R4. Data can be easily linked with generic real-world entities (e.g., locations).

    • The data include geographic information (coordinates, addresses), trip information between stations, as well as the time and distance of each trip.

  • R5. Documentation exists for the data.

    • For bicycle availability, documentation exists regarding the CSV structure, while for the rest, only the metadata of the CSV is available.

  • R6. Data can come from multiple data sources.

    • The files have been obtained from two different sources, the Madrid City Council Open Data Portal and EMT Madrid.

Relations with other datasets

This project aims to analyze and visualize the usage patterns of the bicycle-sharing system in Madrid, leveraging various datasets to gain insights into user behavior, station performance, and overall system efficiency.

  • Bicycle Stations

    Provides information about the location of each station, its status, and the number of available bicycles.

  • Travels

    Provides information about which bicycle made the trip, including the origin and destination stations, as well as the duration of the trip.

  • Bicycles Availability

    Measures the overall availability of bicycles across the entire network, based on usage-related data.
