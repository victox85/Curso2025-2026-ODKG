Dataset analysis

  • bici_disponibilidad20221031.csv

    Number of records in the CSV : 3640

    • DIA

      Date (dd/mm/yyyy)

      This is the temporal key: it allows analyzing daily trends, seasonality, and usage peaks..

    • HORAS_TOTALES_USOS_BICICLETAS

      Total accumulated hours in which the bicycles were in use during that day.

      Useful information: reflects the real usage level of the system and service intensity.

    • FHORAS_TOTALES_DISPONIBILIDAD_BICICLETAS_EN_ANCLAJESID

      Total hours in which bicycles were available at the docks (waiting to be used).

      Useful information: measures system capacity, idle time, or underutilization.

    • TOTAL_HORAS_SERVICIO_BICICLETAS

      Sum of the two previous fields: service time = usage time + availability time.

      Represents the “total supply” that the system offered during that day.

    • MEDIA_BICICLETAS_DISPONIBLES

      Average number of bicycles available at the docks throughout the day.

      Useful information: reflects system coverage and adequacy. A drop may indicate saturation.

    • USOS_ABONADO_ANUAL

      Number of uses by annual subscribers.

      Useful information: identifies frequent/local users.

    • USOS_ABONADO_OCASIONAL

      Number of uses by temporary users (tourists, visitors).

      Useful information: allows comparison between tourism and resident usage.

    • TOTAL_USOS

      Sum of the two previous fields.

      Main demand metric.

  • bikestationbicimad_csv.csv

    Number of records in the CSV : 610

    • FID

      Internal identifier of the row (Feature ID).

      Usually comes from a GIS (Geographic Information System).

    • Shape

      Type of associated geometry, here always Point.

      Means that each station is represented as a point on the map.

    • IdStation

      Unique identifier of the station.

      Primary key used to link with other data (e.g., availability or incidents).

    • Activate

      Binary indicator (1 = active, 0 = inactive).

      Indicates whether the station is operational.

    • Address

      Station address.

      Textual information, useful for geolocation or joining with other street/neighborhood datasets.

    • DockBikes

      Number of bikes currently docked at the station.

      Represents the “available supply” in real time.

    • FreeBases

      Number of free docks at the station.

      Indicates available capacity for returning bikes.

    • PosicionST

      Original coordinates as a string (long lat).

      Redundant since you already have POINT_X and POINT_Y, but useful for debugging.

    • Ligth

      Station light/signal status.

      Usually represents service state:

      • 0 = Free
      • 1 = Busy
      • 2 = Closed / Maintenance

    • Name

      Station name.

      Example: “2 - Metro Callao”. Useful for maps or reports.

    • NoAvailabl

      Boolean/numeric indicator showing if the station is completely out of service (0 = available, 1 = not available).

    • number

      Another identifier (shorter than IdStation, e.g., “2” in “2 - Metro Callao”).

      Can be used as a user-visible code.

    • TotalBases

      Total capacity of the station = DockBikes + FreeBases.

      Shows the size of each station.

    • POINT_X / POINT_Y

      Numeric coordinates in decimal format (longitude, latitude).

      Cleaner than PosicionST for plotting or using in GIS systems.

Resource naming strategy

Namespaces

Ontology namespace: http://example.org/ns# (prefix ns:) for classes and predicates.
Instance namespace: http://example.org/resource/ (prefix ex:) for individuals.

Entity Naming

Each entity has a URI combining a readable type name and a unique identifier or label:

http://example.org/resource/[EntityType]_[IdentifierOrName]
